    家用伺服器折腾手记-3：Mac mini 配置 Nginx 反向代理教程

    前言 📝

    在前两篇文章中，我们已经搭建了基础的 Docker 环境并部署了 MoviePilot。这次我们将探索如何使用 Nginx 反向代理来实现外网访问和 SSL 加密，让我们能够安全地访问家庭服务器上的各种服务。

    准备工作 🛠️

    提示：确保你的 Mac mini 有足够的内存（建议 16GB 以上）和存储空间（建议 256GB 以上）。

    1. 创建工作目录

    mkdir -p ~/docker/homebridge
cd ~/docker/homebridge

    2. 创建配置文件

    # docker-compose.yml
version: '3'
services:
  homebridge:
    image: ghcr.io/homebridge/homebridge:latest
    container_name: homebridge
    restart: unless-stopped
    ports:
      - "8080:8080"
    volumes:
      - ./data:/data

    部署步骤 🚀

    1. 启动服务

    docker-compose up -d

    2. 检查服务状态

    docker ps | grep homebridge
docker logs -f homebridge

    基础配置 ⚙️

    1. 访问 Web 界面

    打开浏览器访问 http://localhost:8080，首次访问需要设置管理员账号和密码。

    进阶配置 🔧

    1. 自定义模型配置

    # modelfile

    性能优化 🚄

    1. 显存管理

    # 在 docker-compose.yml 中添加显存限制
services:
  homebridge:
    environment:
      - CUDA_MEMORY_LIMIT=8G

    2. 模型量化

    # 使用量化版本的模型以减少内存占用
docker exec -it homebridge pull llama2:7b-q4
