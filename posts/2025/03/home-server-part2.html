发布时间：2025-03-08 | 分类：服务器, Docker, 影视

前言 📝

在上一篇文章中，我们介绍了如何在 Mac mini 上搭建 Docker 环境。这次我们将继续探索，使用 Docker Compose 部署 MoviePilot，这是一个强大的电影管理工具，可以帮助我们自动下载和管理电影资源。MoviePilot 是 NAStool 的 Python 重制版，提供了更好的性能和更多的功能。

准备工作 🛠️

提示：确保你已经按照上一篇文章完成了 Docker 环境的搭建。

1. 创建工作目录

mkdir -p ~/docker/moviepilot cd ~/docker/moviepilot

2. 创建配置文件

# docker-compose.yml version: '3.3' services: moviepilot: image: jxxghp/moviepilot:latest container_name: moviepilot hostname: moviepilot network_mode: bridge environment: - PUID=1000 - PGID=1000 - UMASK=022 - TZ=Asia/Shanghai - MOVIEPILOT_AUTO_UPDATE=true - MOVIEPILOT_CN_UPDATE=true - MOVIEPILOT_PROXY_HOST= - MOVIEPILOT_PROXY_PORT= - MOVIEPILOT_PROXY_USERNAME= - MOVIEPILOT_PROXY_PASSWORD= - MOVIEPILOT_SUPERUSER= - MOVIEPILOT_PASSWORD= volumes: - ./config:/config - ./data:/data - /path/to/movies:/movies - /path/to/tv:/tv - /path/to/downloads:/downloads ports: - 3000:3000 restart: unless-stopped

部署步骤 🚀

1. 启动服务

docker-compose up -d

2. 检查服务状态

docker ps | grep moviepilot docker logs -f moviepilot

基础配置 ⚙️

1. 访问 Web 界面

打开浏览器访问 http://localhost:3000，使用配置文件中设置的超级管理员账号密码登录。

2. 配置媒体服务器

MoviePilot 支持多种媒体服务器：

  • Emby
  • Jellyfin
  • Plex

3. 配置下载器

支持的下载器包括：

  • qBittorrent
  • Transmission
  • 115网盘
  • 阿里云盘

4. 配置索引器

添加以下常用索引器：

  • Private Tracker（私有站点）
  • Public Tracker（公开站点）
  • 预配置站点

进阶配置 🔧

1. 订阅规则设置

# 示例订阅规则 - name: "4K电影" type: "电影" resolution: ["4K"] quality: ["WEB-DL", "REMUX"] size: [4000, 50000] - name: "剧集追更" type: "电视剧" resolution: ["1080p"] quality: ["WEB-DL"] size: [500, 3000]

2. 自定义识别词

# 自定义识别词配置 custom_words: - name: "限制级" regex: "NC-17|R级|限制级" replace: "R" - name: "国语配音" regex: "国语|国语配音|中配" replace: "国语"

性能优化 🚄

1. 资源限制

# 在 docker-compose.yml 中添加资源限制 services: moviepilot: deploy: resources: limits: cpus: '2' memory: 2G reservations: cpus: '0.5' memory: 500M

2. 缓存优化

建议：将下载缓存目录挂载到 SSD 上以提高性能。

常见问题 💡

1. 媒体识别问题

  • 检查文件命名是否规范
  • 确认媒体信息是否正确
  • 查看日志排查问题

2. 下载问题

  • 检查下载器连接状态
  • 确认磁盘空间是否充足
  • 验证 Tracker 可用性

进阶使用 🎯

1. API 集成

# 使用 API 获取媒体信息 import requests def get_media_info(api_key, media_id): url = f"http://localhost:3000/api/v1/media/{media_id}" headers = {"X-API-Key": api_key} response = requests.get(url, headers=headers) return response.json()

2. 自动化脚本

# 自动清理脚本 import os import shutil from datetime import datetime, timedelta def clean_old_downloads(): downloads_dir = "/path/to/downloads" threshold = datetime.now() - timedelta(days=30) for root, dirs, files in os.walk(downloads_dir): for file in files: file_path = os.path.join(root, file) file_time = datetime.fromtimestamp(os.path.getmtime(file_path)) if file_time < threshold: try: os.remove(file_path) print(f"已删除: {file_path}") except Exception as e: print(f"删除失败: {file_path}, 错误: {e}") if __name__ == "__main__": clean_old_downloads()
