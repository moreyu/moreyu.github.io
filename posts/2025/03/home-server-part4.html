家用伺服器折腾手记-4：Mac mini 部署 AList 服务 🗂️

发布于 2025-03-10 · Docker 🐳 文件管理 📁 自动化 ⚙️

继续我们的 Mac mini 服务器系列！今天我们要介绍如何使用 Docker 部署 AList，一个强大的文件列表程序。通过 AList，我们可以实现跨设备文件传输、搭建私人影视库，甚至可以整合各种云存储服务。

为什么选择 AList？ 🤔

  • 支持多种存储方式：本地存储、WebDAV、S3、OneDrive 等
  • 美观的界面：支持多种主题，响应式设计
  • 强大的文件预览：支持视频、音频、图片、文档等多种格式
  • 便捷的文件管理：支持上传、下载、复制、移动等操作
  • 完善的权限控制：可以设置用户权限和访问密码

环境准备 🛠️

确保你的 Mac mini 上已经安装了 Docker 和 Docker Compose。如果还没有安装，可以参考我们的第一篇文章。

部署步骤 📝

1. 创建配置文件

首先创建一个新的目录来存放 AList 的配置文件：

mkdir -p ~/docker/alist
cd ~/docker/alist

创建 docker-compose.yml 文件：

version: "3.8"
services:
  alist:
    image: xhofe/alist:latest
    container_name: alist
    volumes:
      - ./data:/opt/alist/data
      - /path/to/media:/media  # 替换为你的媒体文件路径
    ports:
      - "5244:5244"
    environment:
      - PUID=1000
      - PGID=1000
      - TZ=Asia/Shanghai
    restart: unless-stopped
注意：请将 /path/to/media 替换为你实际的媒体文件路径。

2. 启动服务

在配置文件目录下运行：

docker-compose up -d

查看初始密码：

docker exec -it alist ./alist admin

3. 基础配置

  1. 访问 http://你的IP:5244
  2. 使用默认用户名 admin 和上一步获取的密码登录
  3. 进入管理面板，修改密码
  4. 添加存储：
    • 本地存储：直接映射本地文件夹
    • WebDAV：连接其他 WebDAV 服务
    • 云存储：配置各类云存储服务

进阶配置 ⚙️

1. 反向代理设置

如果你想通过域名访问 AList，可以配置 Nginx 反向代理：

server {
    listen 80;
    server_name alist.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:5244;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

2. SSL 配置

建议配置 SSL 证书，确保数据传输安全：

server {
    listen 443 ssl;
    server_name alist.yourdomain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # ... 其他配置同上
}

3. 文件预览优化

AList 支持多种文件预览方式，可以根据需要进行配置：

  • 视频预览：支持 HLS 转码，平滑播放
  • 音频预览：支持在线播放和播放列表
  • 文档预览：支持 Office、PDF 等格式
  • 图片预览：支持图片缩放和幻灯片播放

常见问题 ❓

权限问题
如果遇到文件访问权限问题，请检查：
  • 容器内的 PUID 和 PGID 设置
  • 挂载目录的权限设置
  • SELinux 配置（如果启用）
